## 为什么做这个站点

Proofline 的第一版目标很清楚：把个人信息、文章、项目、论文和作品集放在一个可长期维护的公开入口里。

它不是一个后台系统，也不是一个复杂 CMS。内容只需要上传到服务器上的 Markdown 目录，站点运行时读取并渲染。

| 能力 | 第一版策略 |
| --- | --- |
| 内容维护 | 上传 `.md` 文件 |
| 展示能力 | Markdown 原生语法 |
| 安全边界 | 禁止脚本和事件属性 |
| 部署模式 | Docker + nginx |

> 这个站点的核心不是“功能多”，而是个人表达稳定、可迁移、维护成本低。
