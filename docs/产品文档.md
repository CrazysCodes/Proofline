# Proofline 产品文档

## 1. 产品定位

Proofline 是一个面向个人长期使用的公开主页、博客、作品集与研究材料入口。

它不是传统博客模板，也不是后台型 CMS。它更像一个公开的“能力证据链”：让面试官、合作方、技术同行在很短时间内看到作者是谁、做过什么、写过什么、研究过什么，以及这些内容如何被持续维护。

一句话定位：

> 一个只读、轻量、可长期维护的个人 Profile / Blog / Portfolio / Research Hub。

## 2. 命名说明

Proofline 由 “Proof” 和 “Line” 组成。

- Proof：代表项目、文章、论文、代码、复盘这些可以被验证的个人产出。
- Line：代表时间线、成长线、作品线和持续维护的公开记录。

这个名字强调“用真实材料证明能力”，比普通个人博客更适合面试链接、作品集入口和研究展示场景。

## 3. 核心原则

- 个人门面优先：首页首先回答“我是谁、关注什么、做过什么、能提供什么价值”。
- 只读系统：无后台、无登录、无公网写入接口、无数据库。
- 内容文件化：文章、项目、页面、论文材料都使用 Markdown 文件维护。
- 长期可维护：部署简单，迁移容易，几年后仍能理解和接管。
- 证据密度高：项目、文章、研究材料要服务于可信度，而不是堆列表。
- 安全边界清晰：应用只读内容目录，不把敏感文件放进公开内容目录。

## 4. 目标用户

| 用户 | 核心诉求 | 站点提供的价值 |
| --- | --- | --- |
| 作者本人 | 沉淀文章、项目、研究和经历 | 低维护成本、内容可控、可迁移 |
| 面试官 / 合作方 | 快速判断背景、能力、代码、论文和写作 | 首页信息密度高，证据入口清楚 |
| 技术同行 / 长期读者 | 持续阅读文章、项目复盘和研究笔记 | 清晰的文章列表、标签、归档和 RSS |

## 5. MVP 范围

### 包含

- 首页：个人信息、外链、精选项目、研究入口、精选文章、最近更新。
- 写作页：文章列表、分类筛选、标签筛选。
- 文章详情页：Markdown 渲染、代码块、表格、引用、标签。
- 项目页：项目列表、状态筛选、项目详情。
- 研究页：研究方向、论文材料、研究笔记入口。
- 关于页：个人介绍、方向、联系方式。
- 标签页和归档页。
- RSS、sitemap、robots。
- Docker Compose 部署方式。
- 运行时读取服务器内容目录，上传 Markdown 后刷新即可展示。

### 不包含

- 后台管理系统。
- 登录系统。
- 在线上传接口。
- 数据库。
- 评论系统。
- 多用户权限。
- 在线编辑器。

## 6. 信息架构

| 页面 | 路径 | 目标 |
| --- | --- | --- |
| 首页 | `/` | 建立第一印象，展示关键证据入口 |
| 写作列表 | `/writing` | 展示所有文章，支持分类和标签筛选 |
| 文章详情 | `/writing/[slug]` | 阅读技术文章、研究笔记、项目复盘 |
| 项目列表 | `/projects` | 展示作品集、开源项目和实验项目 |
| 项目详情 | `/projects/[slug]` | 展示问题、贡献、结果、技术栈和链接 |
| 研究页 | `/research` | 展示研究方向、论文和研究笔记 |
| 关于页 | `/about` | 展示个人背景和联系入口 |
| 标签页 | `/tags`、`/tags/[tag]` | 按主题聚合内容 |
| 归档页 | `/archive` | 按时间浏览文章 |
| 订阅 | `/rss.xml` | 提供 RSS 订阅 |

兼容路径：`/posts` 和 `/posts/[slug]` 复用写作页能力。

## 7. 内容模型

### 文章 Post

用于技术文章、项目复盘、研究笔记、服务器记录、读书笔记。

```yaml
title: 重建服务器记录
date: 2026-05-26
description: Debian 初始化、Docker Compose 服务编排和基础设施部署记录
tags: [server, docker, infra]
category: 技术折腾
featured: true
draft: false
```

### 项目 Project

用于作品集、开源项目、实验项目和产品原型。

```yaml
name: Proofline
date: 2026-05-26
description: 一个只读的个人主页、博客、作品集与研究入口。
status: building
tags: [portfolio, nextjs, markdown]
tech: [Next.js, TypeScript, Markdown]
repo: https://github.com/CrazysCodes/Proofline
demo: https://example.com
featured: true
draft: false
problem: 个人内容分散在简历、GitHub、博客和论文链接中，难以形成完整证据链。
contribution: 设计并实现一个只读内容系统，用 Markdown 文件驱动页面展示。
result: 形成可长期维护的个人公开主页和作品集入口。
```

### 论文 / 研究材料 Paper

用于论文、报告、实验材料和研究记录。该目录是可选能力，没有内容时研究页会展示研究方向和研究笔记入口。

```yaml
title: Adaptive Speculative Decoding Notes
authors: Dongjun Hao
venue: Research Note
year: 2026
abstract: 关于推理优化的一组实验记录。
keywords: [llm, inference, evaluation]
links:
  - label: PDF
    url: https://example.com/paper.pdf
featured: true
draft: false
```

## 8. 内容发布方式

内容目录：

```text
content/
  posts/
  projects/
  papers/
  pages/
```

发布流程：

1. 本地编写 Markdown 文件。
2. 通过 SSH、SFTP 或文件同步上传到服务器内容目录。
3. 刷新网站，应用运行时读取内容并展示。

生产部署时，内容目录以只读方式挂载进容器：

```yaml
volumes:
  - ./content:/app/content:ro
```

## 9. MVP 验收标准

- 首页能清楚展示个人定位、链接、项目、研究入口、文章和最近更新。
- 上传文章 Markdown 后，`/writing` 和文章详情页可访问。
- 上传项目 Markdown 后，`/projects` 和项目详情页可访问。
- `/rss.xml`、`/sitemap.xml`、`/robots.txt` 正常输出。
- 应用不提供后台、登录、上传或数据库能力。
- 容器部署时内容目录只读挂载。
- 生产构建通过。

## 10. 后续迭代规划

### P0：真实内容替换

- 替换真实姓名、头像缩写、个人介绍、GitHub、邮箱、CV、论文链接。
- 补充 3 个代表项目。
- 补充 3 篇高质量文章或项目复盘。
- 补充关于页真实经历。

### P1：阅读体验增强

- 文章详情页增加目录。
- 增强代码高亮。
- 增加上一篇 / 下一篇导航。
- 增加项目详情中的截图或封面能力。

### P2：检索与组织

- 增加站内全文搜索。
- 标签页支持项目、文章、研究材料混合筛选。
- 归档页支持项目和论文材料。

### P3：公开传播能力

- 增加 Open Graph 图片生成。
- 增加简历式单页导出。
- 增加精选内容的手动排序字段。
- 增加内容健康检查脚本，提前发现缺失字段和无效链接。
